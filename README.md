# 排序演算法效能與複雜度分析報告

## 一、 五個排序法介紹

以下介紹本次實驗分析的五種經典排序演算法，其基本原理與操作方式：

1. **氣泡排序法 (Bubble Sort)**
   * **基本原理**：透過相鄰元素的比較與交換，讓較大（或較小）的元素如同水中的氣泡一樣，逐漸浮到陣列的頂端（末端）。
   * **操作方式**：從陣列開頭開始，每次比較相鄰的兩個元素。如果左邊比右邊大就交換位置。每一輪掃描都會把當前未排序區塊的最大值推到正確的最後方，直到整趟掃描沒有發生任何交換為止。

2. **選擇排序法 (Selection Sort)**
   * **基本原理**：每次從尚未排序的資料中找出最小值，並將其與未排序區塊的第一個元素交換，藉此不斷擴大「已排序區塊」。
   * **操作方式**：將陣列視為「已排序」與「未排序」兩部分。每一回合掃描未排序區塊尋找最小值，找到後將它與未排序區塊的最前端元素互換，直到所有元素都就位。

3. **插入排序法 (Insertion Sort)**
   * **基本原理**：如同打撲克牌整理手牌，將尚未排序的元素逐一抽出，並插入到左側已經排序好的序列中適當的位置。
   * **操作方式**：將陣列的第一個元素視為已排序。接著從第二個元素開始，向左邊已排序的區塊逐一比較，若左邊較大則將其向右平移，直到找到不比自己大的元素，即將數值插入該空位。

4. **合併排序法 (Merge Sort)**
   * **基本原理**：採用「分治法 (Divide and Conquer)」策略，將大問題拆解成小問題分別解決後，再將結果合併。
   * **操作方式**：將陣列不斷對半分割，直到每個子陣列只剩下一個元素（視為已排序）。接著將這些子陣列兩兩合併，合併過程中進行大小比較，最終整合成一個完整的已排序陣列。

5. **堆積排序法 (Heap Sort)**
   * **基本原理**：利用「二元樹」的邏輯結構（堆積樹）來尋找最大值或最小值，是一種不需要大量額外記憶體的高效排序法。
   * **操作方式**：首先將陣列轉換為「最大堆積樹 (Max Heap)」，此時根節點必為最大值。將根節點與陣列最後一個元素交換（將最大值排好），然後將剩餘的節點重新調整為最大堆積樹，重複此動作直到排序完成。

## 二、 複雜度分析

不同演算法在處理資料時，所花費的時間（時間複雜度）與額外佔用的記憶體（空間複雜度）各有不同。下表為五種排序法的理論複雜度分析：

| 排序演算法 | 最佳時間複雜度 | 平均時間複雜度 | 最壞時間複雜度 | 空間複雜度 | 穩定性 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **氣泡排序** | $\mathcal{O}(n)$ | $\mathcal{O}(n^2)$ | $\mathcal{O}(n^2)$ | $\mathcal{O}(1)$ | 穩定 |
| **選擇排序** | $\mathcal{O}(n^2)$ | $\mathcal{O}(n^2)$ | $\mathcal{O}(n^2)$ | $\mathcal{O}(1)$ | 不穩定 |
| **插入排序** | $\mathcal{O}(n)$ | $\mathcal{O}(n^2)$ | $\mathcal{O}(n^2)$ | $\mathcal{O}(1)$ | 穩定 |
| **合併排序** | $\mathcal{O}(n \log n)$ | $\mathcal{O}(n \log n)$ | $\mathcal{O}(n \log n)$ | $\mathcal{O}(n)$ | 穩定 |
| **堆積排序** | $\mathcal{O}(n \log n)$ | $\mathcal{O}(n \log n)$ | $\mathcal{O}(n \log n)$ | $\mathcal{O}(1)$ | 不穩定 |

> **說明**：
> * **穩定性**：若兩個元素數值相同，排序後它們的相對位置是否保持不變。
> * 氣泡排序與插入排序在資料「已接近排序完成」時，最佳時間複雜度可達 $\mathcal{O}(n)$。
> * 合併排序需要額外 $\mathcal{O}(n)$ 的陣列空間來存放合併結果。

## 三、 數據模擬

為了驗證理論，針對不同規模的資料量 $n$ 進行模擬實驗。實驗條件假設為**隨機生成的整數陣列**，測量演算法完成排序所需的執行時間。

* **小規模資料**：$n = 1,000$
* **中規模資料**：$n = 10,000$
* **大規模資料**：$n = 100,000$

*(註：以下為基於現代標準處理器運算能力的模擬估算耗時，實際執行時間會依硬體與程式語言實作略有浮動。)*

## 四、 結果呈現與比較

### 1. 執行時間實驗數據 (單位：毫秒 ms)

| 演算法 / 資料量 ($n$) | $n = 1,000$ | $n = 10,000$ | $n = 100,000$ | 成長趨勢 |
| :--- | :--- | :--- | :--- | :--- |
| **氣泡排序 (Bubble)** | 2.5 ms | 230 ms | 22,500 ms | 二次方成長 |
| **選擇排序 (Selection)** | 1.8 ms | 150 ms | 14,200 ms | 二次方成長 |
| **插入排序 (Insertion)** | 1.2 ms | 85 ms | 8,300 ms | 二次方成長 |
| **合併排序 (Merge)** | 0.2 ms | 2.5 ms | 31 ms | 對數線性成長 |
| **堆積排序 (Heap)** | 0.3 ms | 3.2 ms | 38 ms | 對數線性成長 |

### 2. 效能成長趨勢比較

當資料量 $n$ 擴大 10 倍（從 10,000 變成 100,000）時：
* **$\mathcal{O}(n^2)$ 家族**（氣泡、選擇、插入）：時間大約暴增了 **100 倍** ($10^2$)。例如氣泡排序從 0.2 秒飆升到 22.5 秒，對於大型資料處理顯得極度吃力。
* **$\mathcal{O}(n \log n)$ 家族**（合併、堆積）：時間大約只增加了 **12 到 15 倍**。即便處理十萬筆資料，依然能在 50 毫秒內瞬間完成。

## 五、 結論與應用建議

根據理論分析與數據模擬結果，可以得出以下具體結論與選擇建議：

1. **處理大規模資料首選「合併排序」或「堆積排序」**：
   當資料量龐大時，$\mathcal{O}(n \log n)$ 的演算法具有壓倒性的優勢。若記憶體空間充足且需要「穩定排序」，請選擇**合併排序**；若環境對記憶體限制嚴格（例如嵌入式系統），則 $\mathcal{O}(1)$ 空間複雜度的**堆積排序**是最佳選擇。

2. **小規模或幾乎已排序的資料適合「插入排序」**：
   雖然插入排序的平均複雜度是 $\mathcal{O}(n^2)$，但它的常數項極小。當 $n$ 很小，或者陣列已經「大致上排好序」時，插入排序的效能會逼近 $\mathcal{O}(n)$，甚至比高階排序法還要快。這也是為什麼許多內建函式庫（如 Python 的 Timsort 或 C++ 的 std::sort）在切割到小區塊時，會自動切換成插入排序的原因。

3. **「氣泡排序」與「選擇排序」多用於教學與特殊場景**：
   兩者的綜合效能在實務上通常最差。氣泡排序主要用於理解資料交換邏輯，而選擇排序的唯一優勢在於它的「交換次數」最少（最多 $n-1$ 次），若寫入硬碟的成本極高時，偶爾會有奇效。
